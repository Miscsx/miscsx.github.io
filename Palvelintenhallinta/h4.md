A)
  -
- Postgresql asennus ja käyttäjän teko

        sudo apt-get -y install postgresql
- Tämän jälkeen tarkistin, että postgres on päällä

        systemctl status postgresql
- Sitten vielä käyttäjän luonti

        sudo -u postgres createdb (whoami)
        sudo -u postgres createuser (whoami)
- Näin on tehty database ja käyttäjä

B)
  - Postgres asennuksen automatisointi
  - Tein ensin uuden roolin

        mdir roles/postgres/tasks -p
- Sitten tein main.yml jonka sisälle lisäsin tämän

        - name: Install postgresql
          apt:
            name: postgresql
            state: present
            update_cache: yes

       - name: Start postgresql
         service:
           name: postgresql
           state: started
           enabled: true
- lisäsin roolin site.yml ja muiden siellä olevien roolien eteen lisäsin # jotta site.yml ajaa vain postgres roolin.
- playbook ajoi tehtävän ja tuli changed 0, koska postgres on jo asennettu ja päällä kokeilin sitten lopettaa postgressin ja ajaa playbookin uudestaan

        sudo systemctl stop postgresql
        ansible-playbook site.yml -K
- Näin sain changed 1, joka oli postgressin käynnistys (started)

C)
  -
- Lisäsin tämän postgres/tasks/main.yml sisälle

      - name: Set listen_addresses
        lineinfile:
          path: /etc/postgresql/14/main/postgresql.conf
          regexp: '^#?listen_addresses'
          line: "listen_addresses = '*'"
      notify: Restart PostgreSQL
- Ja tein handlersin postgressin alapuolelle eli /postgres/handlers/main.yml ja sen sisään tämän

      - name: Restart PostgreSQL
        service:
          name: postgresql
          state: restarted
- Ajoin ansible-playbookilla ja tarkistin vielä komennolla

      cat /etc/postgresql/17/main/postgresql.conf | grep listen_addresses

D) ja E)
  -
- Ajoin teron esimerkistä komennot

      sudo apt-get purge -y postgresql
      sudo rm -rf /etc/postgresql
- Eli poistin postgressin ja konfiguraation
- Ajoin ansible-playbookin uudestaan
- Ensimmäisellä ajolla tuli 1 failed koska, nähtävästi jokin postgressin cluster tiedosto oli korruptoitunut
- Tämän korjasin käsin

      sudo systemctl stop postgresql
      sudo rm -rf /var/lib/postgresql/17/main
      sudo rm -rf /etc/postgresql/17/main
      sudo pg_createcluster 17 main --start
- Tarkistin sen jälkeen vielä, että korruptoitunut tiedoston on paikallaan

      pg_lsclusters
- Nyt kaikki toimi
- Ansible-playbook antaa nyt kaikki ok
