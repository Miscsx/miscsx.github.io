- Ansible Apache voidaan asentaa playbookilla
- Playbook ei tee muutoksia jos kaikki on kunnossa
- Palvelu käynnistetään ja asetetaan automaattisesti käynnistyväksi

Ansible Community Documentation
-
- Handlerit suoritetaan, jos jokin tehtävä aiheuttaa muutoksen
- Tyypillisesti käyttö on palvelun uudelleenkäynnistys konfig muutoksen jälkeen
- Estää turhat uudelleenkäynnistykset
- Task ilmoittaa handlerille notify-avainsanalla
- Handler ajetaan playbookin lopussa
- Handler suoritetaan kerran viakka notify tulisi monta kertaa

Ansible-doc service
-
- name = palvelun nimi
- state = tila esim (started tai stopped)
- enabled = käynnistyykö kun boottaa
- Examples

    - Palvelun käynnistys
    - Automaattikäynnistyksen asettaminen
    - Uudelleenkäynnistys

A)
  -
- Apachen asennus
  
      sudo apt update
      sudo apt install apache2 -y
- Käynnistys

      sudo systemctl start apache2
      sudo systemctl enable apache2
- Sivusta muokattava käyttäjälle

      sudo chown $USER:$USER /var/www/html/index.html

B)
  -
- Apachen sammuttaminen

      sudo systemctl stop apache2
      sudo system disable apache2
- Nginx asennus

      sudo apt install nginx -y
- Käynnistys

      sudo systemctl start nginx
      sudo systemctl enable nginx
- Muokattava sivu

      sudo chown $USER:$USER /var/www/html/index.nginx-debian.html

C)
  - 
- H2 tehtävässä tein uuden roolin joka lataa nginx:in

      roles/nginx/tasks/main.yml
- Lisäsin tänne main.yml tiedostoon nämä

      - name: Start nginx
        service:
          name: nginx
          state: started
          enabled: yes
- Sitten vain ajettiin ansible-playbook
